# UI增强功能需求

## 概述
为IPTV应用添加用户体验改进功能，包括播放列表管理、加载状态提示和全屏播放支持。

## 功能需求

### 1. 播放列表名称编辑
**需求ID**: REQ-1
**优先级**: 高

添加Xtream和M3U播放列表时，允许用户自定义播放列表名称：
- 在添加对话框中增加可选的"名称"输入字段
- 如果用户不输入名称，使用默认名称（M3U使用URL，Xtream使用服务器地址）
- 名称字段应该在URL/服务器信息之前显示
- 支持中文输入

### 2. 添加播放列表时的加载提示
**需求ID**: REQ-2
**优先级**: 高

在添加Xtream和M3U播放列表时显示加载动画：
- 点击"添加"按钮后显示加载对话框
- 显示加载进度指示器（CircularProgressIndicator）
- 显示当前操作状态文本（如"正在连接..."、"正在加载频道..."）
- 加载成功后自动关闭对话框
- 加载失败时显示错误信息，允许用户重试或取消

### 3. 播放器加载状态提示
**需求ID**: REQ-3
**优先级**: 高

打开直播链接时，在等待网络响应期间显示加载动画：
- 在PlayerScreen中检测BUFFERING状态
- 显示居中的加载动画覆盖层
- 显示"正在加载..."文本
- 加载动画应该半透明，不完全遮挡视频区域

### 4. 手动刷新播放列表
**需求ID**: REQ-4
**优先级**: 中

为Xtream和M3U播放列表提供手动刷新功能：
- 在PlaylistScreen的每个播放列表项上添加刷新按钮
- 点击刷新按钮重新加载该播放列表的内容
- 刷新时显示加载指示器
- 刷新成功后更新播放列表数据
- 刷新失败时显示错误提示

### 5. 全屏播放支持
**需求ID**: REQ-5
**优先级**: 高

支持视频全屏播放：
- 在PlayerScreen中添加全屏切换按钮
- 支持通过按钮切换全屏/窗口模式
- Desktop平台使用VLC的全屏API
- Android平台实现系统UI隐藏和全屏布局
- 全屏时隐藏TopAppBar
- 退出全屏时恢复正常布局

## 非功能需求

### 性能要求
- 加载动画应该流畅，不影响后台操作
- 刷新操作应该异步执行，不阻塞UI

### 用户体验要求
- 所有文本使用中文
- 错误信息应该清晰易懂
- 加载状态应该及时反馈给用户

### 兼容性要求
- 支持Android和Desktop平台
- 保持现有功能不受影响
